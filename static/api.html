Artikel-Interface

Artikel können vollständig über die API abgefragt werden. Die Schnittstelle erlaubt Volltextsuche und Facetierung und gibt eine JSON-Repräsentation der Artikelbeschreibung zurück.

Beispiel: /v1/spon/article?limit=3&facet=authors_name&q=Kabinett — Drei Artikel mit dem Text "Kabinett", sowie eine Liste der Autoren, die den Begriff "Kabinett" in den meisten Artikeln verwendet haben.

Unterstützte Parameter

  • limit: Maximale Anzahl der gelieferten Ergebnisse. Es werden höchstens 500 Artikel pro Anfrage geliefert.
  • offset: Offset der zurückgelieferten Artikel zur Paginierung.
  • q: Freitext-Anfrage, kann die Lucene Query Syntax verwenden. Alle Felder einer Artikel-Repräsentation werden indiziert, geschachtelte Strukturen sind durch einen Unterstrich abrufbar.
  • facet: Facette abfragen. Eine Facette fasst die Anzahl der Treffer zu jedem Wert eines einzelnen Felds zusammen. facet-limit beschränkt die Anzahl der abgerufenen Werte.
  • sort: Sortierung, sinnvoll sind entweder score desc (Relevanz zur Suchanfrage) oder launchdate desc (Veröffentlichungstermin, Standardeinstellung).
  • filter: Anfragefilter, Syntax entspricht der von q. Filter können verwendet werden, um q einfach zu halten.

Einzelne Artikel

Einzelne Artikel können über ihre ID direkt abgerufen werden: /v1/spon/article/888628. Die Representation als Einzelergebnis ist umfangreicher als die Listenansicht und umfasst z.B. Autoren und Themenzuordnungen.
